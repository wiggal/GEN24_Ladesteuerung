💾 Installation Dockerimage Gen24-Ladesteuerung im Synology-Docker:

(Inhalte teilweise von @jaydee73 danke 👍) Stand: 2026-05-30

Ordner für Docker-Container anlegen und Dateien bereitstellen

In einem Ordner nach Wahl auf der Synology einen entsprechenden Ordner für den gen24-Container anlegen.
Am besten unter dem bereits vorhandenen Root-Ordner „docker“ evtl. in einem Verzeichnis gen24_ladesteuerung. Aus dem gen24-Github-Repo folgende Dateien in den angelegten Ordner kopieren:

XY_priv.ini Dateien nach eigenen Bedürfnissen anpassen (siehe 4_Hilfe.html und allgemeine Installationshinweise)
config_priv.ini (aus dem Unterordner html), nach eigenen Bedürfnissen anpassen (zumindest KWp-Wert anpassen)

Datei Crontab.log neu erzeugen (bleibt zunächst leer, wird dann vom Script gefüllt).

Container erstellen

Container-Manager starten --> Container --> Erstellen

  • Fenster: Allgemeine Einstellungen

Als Image „Image hinzufügen“ auswählen
Nach wiggal/gen24_ladesteuerung suchen -> Download --> Tag auswählen: latest
Container-Name frei wählen z.B.: GEN24_ladesteuerung
Automatischen Neustart aktivieren --> WEITER

  • Fenster: Erweiterte Einstellungen

Porteinstellungen ->> + Hinzufügen lokaler Port 2424 Container-Port 2424 TCP
In den Volume-Einstellungen folgende Dateien mappen:
+ Datei hinzufügen --> QUELLE auswählen --> ZIEL eingeben
ACHTUNG: Alle Quelldateien müssen als Datei existieren!!

QUELLE ZIEL
xy_priv.ini /home/GEN24/CONFIG/xy_priv.ini
Prog_Steuerung.sqlite /home/GEN24/CONFIG/Prog_Steuerung.sqlite
config_priv.ini /home/GEN24/html/config_priv.ini
Crontab.log /home/GEN24/Crontab.log
PV_Daten.sqlite /home/GEN24/PV_Daten.sqlite
weatherData.sqlite /home/GEN24/weatherData.sqlite

Im Bereich Netzwerk noch bridge auswählen, sofern nicht schon voreingestellt.
Alle anderen Einstellungen bleiben unverändert.
WEITER

  • Fenster: Zusammenfassung

Alles nochmal prüfen. Container beim Erstellen starten auswählen.
FERTIG

Nach dem Start im Protokoll nachschauen, ob Fehlereinträge auftauchen.

Aufruf der Oberfläche im Browser unter:
Synology_name_oder_ip:2424

Update (wird im Container-Manager angezeigt)

Wenn eine gemappte Datei (z.B. config.ini oder config.ini) geändert werden muss:

  • Container stoppen
  • Gemappte Datei ändern
  • Updateprozess des Container-Images über den Container Manager anstoßen
  • Container starten

Wenn sich nur das Image geändert hat:
Dann klickt man einfach auf updaten.
