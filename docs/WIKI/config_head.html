Softwareupdate bei einem Git-Repositiry:

In der Überschrift wird die installierte Version der Ladesteuerung angezeigt.
Wenn das Quellverzeichnis ein Git-Repository ist, also mit "git clone https://github.com/wiggal/GEN24_Ladesteuerung.git ." erzeugt wurde (siehe Install im WIKI),
wird neben der Überschrift ein Updatebutton angezeigt. Ist eine neuere Version auf Github vorhanden ist, kann über diesen Button ein Update gemacht werden. 

Beispielausgaben währen des Update:

Neue Version: v0.40.1

✅ Configdateien seit dem letzten Update, ohne Veränderung!✅

oder:

⚠️ Differenzen in den Configdateien seit dem letzten Update
evtl. die entsprechenden _priv.ini-Dateien anpassen!⚠️ 

	
Differenzen in den Configdateien seit dem letzten Update.
evtl. die entsprechenden _priv.ini-Dateien ergänzen!

  +; print_level steuert .... (EnergyController.py)
  -[gen24]
  +[inverter]
  +; InverterTyp für Fronius Gen24 und Verto = gen24
  +InverterTyp = gen24

Update der _priv.ini Dateien▲

In dem Dropdown können alle vorhandenen _priv.ini-Dateien ausgewählt und angezeigt werden.
Anschließend kann die ausgewählte _priv.ini-Datei editiert oder wenn eine originale ini-Datei vorhanden ist mit dieser verglichen und auf deren Stand upgedatet werden.

Folgende Optionen sind beim Update möglich:

Werte aus XY_priv.ini


Die Werte, die in der  XY_priv.ini den Variablen zugewiesen wurden. Sie können hier, genauso wie unter editieren, verändert werden.

Veränderter Kommentar aus XY.ini


Kommentare, die sich in der original XY.ini verändert haben, werden hier in die XY_priv.ini übernommen.

Fehlt in XY.ini


Werte die der XY_priv.ini hinzugefügt wurden fehlen in der original XY.ini.
Wenn die der XY_priv.ini hinzugefügten Werten erhalten bleiben sollen, müssen die Optionsfelder angewählt bleiben.
Die Werte, die ursprünglich aus der XY.ini übernommen wurden, dort aber weggefallen sind (weil nicht mehr benötigt), können durch Abwahl der Optionsfelder auch aus der XY_priv.ini entfernt werden.

Fehlt in XY_priv.ini


Werte die in der XY.ini enthalten sind,aber in der XY_priv.ini fehlen, werden der XY_priv.ini hinzugefügt, wenn die Optionsfelder ausgewählt bleiben.
